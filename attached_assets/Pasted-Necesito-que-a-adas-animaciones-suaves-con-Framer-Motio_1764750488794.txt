Necesito que añadas animaciones suaves con Framer Motion a la landing "Rio Trip Vibes" sin romper nada existente. Las animaciones deben activarse con el scroll y repetirse cada vez que el usuario vuelva a la sección.

## IMPORTANTE: Patrón técnico correcto

Usa SIEMPRE este patrón (no uses whileInView directamente porque falla en producción):

import { useRef, useEffect } from "react";
import { motion, useInView, useAnimation } from "framer-motion";

// Dentro del componente:
const ref = useRef(null);
const controls = useAnimation();
const inView = useInView(ref, { once: false, amount: 0.15 });

useEffect(() => {
if (inView) {
controls.start("visible");
} else {
controls.start("hidden");
}
}, [inView, controls]);

// En el JSX:
<motion.div
ref={ref}
initial="hidden"
animate={controls}
variants={miVariante}

## 1) Hero - Parallax cinematográfico

**Qué hacer:**
- El fondo del hero debe tener parallax vertical suave que funcione siempre al hacer scroll
- Usa `useScroll` con `layoutEffect: false` (crítico para producción)
- Transforma el scrollY a movimiento vertical de 0 a 250px en 1000px de scroll
- Escala de 1.1 a 1.3 para efecto de profundidad
- El texto, botones y navbar NO se mueven, solo el fondo

**Animación de entrada:**
- Título: desde opacity 0, y: 20 hasta opacity 1, y: 0 en 0.7s
- Subtítulos: igual pero con delay de 0.3s y 0.4s para stagger
- Botones: desde scale 0.9 hasta scale 1 con delay de 0.5s y 0.6s

**Código del parallax:**
const { scrollY } = useScroll({ layoutEffect: false });
const backgroundY = useTransform(scrollY, , );
const backgroundScale = useTransform(scrollY, , [1.1, 1.3]);

## Reglas de oro

1. **NUNCA uses `whileInView` directamente** → Usa el patrón `useAnimation` + `useEffect`
2. **Siempre `once: false`** para que se repita
3. **`layoutEffect: false`** en useScroll (crítico)
4. **amount entre 0.08-0.15** para trigger temprano
5. **Respeta `prefersReducedMotion`** con `reducedMotionVariants`
6. **No rompas responsive** ni el menú fijo
7. **Solo transform y opacity** en las animaciones

Al hacer scroll:

Nunca debe haber elementos "atascados" en estado hidden. Si ves que algo no aparece, es porque falta el patrón useAnimation + useEffect correctamente implementado.
